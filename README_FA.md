# EasyOAuth  
یک لابراتور آسیب‌پذیر OAuth 2.0 برای نمایش ضعف‌های واقعی در فرآیند Authorization.

در بسیاری از مناطق، دسترسی به پلتفرم‌های آموزشی بین‌المللی و لابراتورهای امنیتی به دلیل محدودیت‌های اینترنتی دشوار یا حتی غیرممکن است. در نتیجه، بسیاری از دانشجویان و علاقه‌مندان امنیت نمی‌توانند لابراتورهای واقعی را تمرین کنند یا مهارت عملی خود را تقویت نمایند.

**EasyOAuth** دقیقاً برای حل همین مشکل ساخته شده است.

هدف این پروژه ارائه یک لابراتور امنیتی کاملاً **محلی** و **قابل اجرا بدون اینترنت** است تا دانشجویان، پژوهشگران و تست‌نفوذکاران بتوانند آسیب‌پذیری‌های OAuth را بدون نیاز به اینترنت پایدار تمرین کنند.

با اجرای این لابراتور به‌صورت محلی، کاربران می‌توانند سناریوهای واقعی حمله را شبیه‌سازی کرده و یاد بگیرند که چگونه یک تنظیم اشتباه کوچک در OAuth می‌تواند به مشکلات جدی مانند **تصاحب حساب (Account Takeover)** منجر شود.

البته این پروژه هنوز در مراحل اولیه است و **ممکن است دارای باگ‌ها، محدودیت‌ها یا بخش‌هایی باشد که نیاز به بهبود داشته باشند**. ما کاملاً از این موضوع آگاهیم و قصد داریم نسخه‌ها و لابراتورهای آینده را با دریافت بازخورد کاربران، **بهینه‌تر و حرفه‌ای‌تر** کنیم.

این پروژه فقط شروع کار است.

ما قصد داریم لابراتورهای آسیب‌پذیر و چالش‌های امنیتی بیشتری بسازیم و منتشر کنیم تا همه—even با اینترنت محدود—بتوانند مهارت‌های امنیتی خود را تقویت کنند.

اگر این پروژه برای یادگیری شما مفید بود، حمایت شما می‌تواند کمک بزرگی برای ادامه توسعه لابراتورهای آموزشی باشد.

# 🛡️ EasyOAuth — لابراتوار آسیب‌پذیری OAuth 2.0

یک محیط آموزشی برای نشان دادن اینکه چگونه پیاده‌سازی اشتباه OAuth 2.0 می‌تواند منجر به آسیب‌پذیری‌هایی مانند **تصاحب حساب** شود.

این لابراتور شامل دو سرویس است:

- Application (Client)  
- OAuth Provider (Authorization Server)

هدف این پروژه شبیه‌سازی یک سناریوی واقعی از یک تنظیم اشتباه OAuth است که در آن مهاجم می‌تواند بدون داشتن اطلاعات ورود، خود را جای **ادمین** جا بزند.

# 📝 تنظیم فایل Hosts

برای اجرای صحیح لابراتور، این دامنه‌ها را به فایل hosts اضافه کنید:

```
tajanapplication.com
tajanprovider.com
```

هر دو باید به این آدرس اشاره کنند:

```
127.0.0.1
```

## لینوکس

فایل hosts را ویرایش کنید:

```bash
sudo nano /etc/hosts
```

افزودن:

```
127.0.0.1 tajanapplication.com
127.0.0.1 tajanprovider.com
```

---

## ویندوز

فایل زیر را با دسترسی Administrator باز کنید:

```
C:\Windows\System32\drivers\etc\hosts
```

افزودن:

```
127.0.0.1 tajanapplication.com
127.0.0.1 tajanprovider.com
```

# 🚀 راه‌اندازی خودکار (پیشنهادی)

اجرای فایل اصلی:

```bash
python3 main.py
```

این اسکریپت به‌صورت خودکار:

- سیستم‌عامل شما را شناسایی می‌کند  
- محیط مجازی venv می‌سازد  
- پکیج‌ها را نصب می‌کند  
- از شما درباره دسترسی به PyPI می‌پرسد  
- در صورت نیاز از Mirror استفاده می‌کند  
- هر دو سرویس را اجرا می‌کند  
- و امکان خاموش کردن هر دو سرویس با دستور:

```
finish
```

را فراهم می‌کند.

لینوکس:
```bash

```

ویندوز:
```bash

```

---

## 3. نصب پکیج‌ها

```bash
pip install -r requirements.txt
```

---

# ❗ اگر به PyPI دسترسی ندارید

در صورت محدودیت اینترنت، از این Mirror استفاده کنید:

```bash
pip install --trusted-host mirror-pypi.runflare.com -i https://mirror-pypi.runflare.com/simple/ -r requirements.txt
```

آپدیت pip از طریق همان Mirror:

```bash
pip install --upgrade pip --trusted-host mirror-pypi.runflare.com -i https://mirror-pypi.runflare.com/simple/
```
